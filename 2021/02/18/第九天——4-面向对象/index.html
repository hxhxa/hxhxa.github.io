第九天——4.面向对象

本文最后更新于：2021年2月23日 凌晨

一、学习面向对象的三条主线

一）Java类及类的成员

属性、方法、构造器；

代码块、内部类

二）面向对象的三大特征

封装性、继承性、多态性、（抽象性）

三）其他关键字

this、super、static、abstract、interface、package、import

“大处着眼，小处着手”

二、面向对象的编程思想

一）面向过程（pop–procedure oriented programming）与面向对象（oop–procedure oriented programming）

二者都是一种思想，面向对象是相对于面向过程而言的。

1.面向过程

面向过程，强调是功能行为，以函数为最小单位，考虑怎么做。

①打开冰箱，②抬起大象放进冰箱，③关闭冰箱门

2.面向对象

面向对象，将功能封装进对象，强调了具备了功能的对象，以类/对象为最小单位，考虑谁来做。面向对象更加强调运用人类在日常的思维逻辑中采用的思想方法与原则，如抽象、分类、继承、聚合、多态等。

二）面向对下个的三大特征

封装（Encapsulation）

继承（Interitance）

多态（Polymorphism）

三）面向对象的思想概述

程序员从面向过程的执行者转化成了面向对象的指挥者

2.面向对象分析方法分析问题的思想和步骤：

1、根据问题需要、选择问题所针对的现实世界中的实体

2、把抽象的实体用计算机语言进行描述，形成计算机世界中类的定义，及借助某种程序语言，把类构造成计算机能够识别和处理的书记结构。

3、将类实例化成计算机中的对象，对象是计算机世界中解决问题的最终工具。=有·

三、面向对象的基本元素：类和对象

一）面向对象的思想概述

1、类（Class）和对象（Object）是面向对象的核心概念

1）类是一类事物的描述，是抽象的、概念上的意义

2）对象是实际存在的该类事物的每个个体，因而也称为实例（instance）

万事万物皆对象

2、面向对象的思想概述

1.类 = 抽象概念的人；对象 = 实实在在的某个人；

2.面向对象程序设计的重点是类的设计

3.类的设计其实就是类成员的设计

二）Java类及类的成员

1.Java代码的世界是由诸多个不同功能的类构成的。

2.常见的类成员

属性：对应类中的成员变量（field = 域 = 字段）

行为：对应类中的成员方法（方法 = 成员变量 = 函数 = method）

创建类的对象 = 类的实例化 = 实例化类

调用对象的结构：属性、方法

调用对象的属性：“对象.属性”

3.类和对象的使用（面向对象思想的实例）

1.创建类，设计类成员（是否具有生命力，是否强大）

2.创建类的对象

3.通过“对象.属性”或”对象.方法”调用对象的结构

三）对象的创建和使用

如果创建了一个类的多个对象，则每个对象都独立的拥有一套类的属性。（非static的）意味着：如果我们修改一个对象的属性值a，则不影响另外一个属性a的值

四、类中属性的使用

属性（成员变量） 局部变量
不同点
在类中声明的位置的不同 直接定义在类的一对{}内 声明在方法内、方法形参、代码块内、构造器形参、构造器内部的变量
关于权限修饰符 可以在声明属性时，指明其权限，使用权限修饰符。常用的权限修饰符：private、public、缺省、protected（封装性）。目前，声明属性时，都使用缺省就可以了。 不可以使用权限修饰
默认初始化值的情况 类的属性根据其类型，都有默认化初始值。整型（byte、short、int）=0；浮点型（float、double）=0.0；字符型（char）0或\u000;布尔型（boolean）false；引用数据类型（类、接口、数组）null 没有默认初始值。意味着在调用局部变量之前，一定要显式赋值。特别地，形参在调用时，我们赋值即可
在内存中加载位置 加载到堆空间中（非static） 加载到栈空间
相同点 1.定义变量的格式一样：数据类型 变量名 = 变量值
2.先声明，后使用
3.变量都有其对应的作用域

3.说明

1）关于权限修饰符

（默认权限先都使用public）

Java规定的4种权限修饰符：private、public、缺省、protected（封装性详细讲）

2）返回值类型（有返回值/没有返回值）

1.如果方法有返回值，则必须在方法声明时，指定返回值的类型。同时，方法中需要使用return关键字来返回指定类型的变量或常量。

如果方法没有返回值，则声明方法时，用void来表示。通常，没有返回值的方法中，不使用return。如果使用return，则表示结束此方法。只能使用“return；”

2.我们定义方法该不该用返回值
